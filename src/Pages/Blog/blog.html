Structure

Our code repository is a well-organized park with different sections and programming language groves. To keep this park green and vibrant, let's abide by these guidelines:

Folder Naming:

  • Give your folders meaningful, descriptive names.
  • No spaces or special characters, please. Keep it all lowercase.
  • Example: "this_is_a_folder"

Code File Naming:

  • Your code files should tell a story. Use clear, lowercase names without spaces or special characters.
  • Avoid generic names like temp.txt or test.c.
  • Example: "this_is_a_file.cpp"

Code Placement

Navigating through our coding park is a breeze. Follow these paths for harmonious coding:

Existing Folder:

  • Respect the ecosystem. If a relevant folder exists, don't plant a new one.
  • File Placement: Nestle your code file directly inside the existing folder.
  • Create a new folder only if the topic remains unexplored.
  • NO NESTED FOLDERS, please.

Coding Style

In our world, coding is an art. Feel free to express yourself! Yet, let's add a touch of discipline to ensure bug-free masterpieces:

  • Craft your code in a separate class or function. Save the main function for testing purposes.
  • Steer clear of typedef or define macros. We want your code to be a standalone masterpiece.
  • Embrace large data typesâ€”long long or double over int or float. A safeguard against unexpected bugs for larger arguments.
  • Let snake_case be your naming convention.
